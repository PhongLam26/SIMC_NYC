## Ablation Configurations

All ablations use the chronological target-week split, validation-selected thresholds, raw-score PR-AUC/ROC-AUC, and a 300,000-row stratified training sample:

- Core history + identifiers
- Core + calendar
- Core + calendar + weather
- Core + calendar + weather + OSM
- Core + calendar + weather + PLUTO
- Full context

Feature-set composition was checked against `data/processed/_model_summaries/inspect_final_dataset_feature_sets.json` and `code_model/inspect_final_dataset.py`. The old `Historical demand only` label was replaced with `Core history + identifiers` because the set includes complaint category, borough, current `complaint_count`, lag/rolling demand features, and history-availability flags. Calendar and weather are cumulative additions; OSM and PLUTO are evaluated separately after weather; full context includes both OSM and PLUTO.
